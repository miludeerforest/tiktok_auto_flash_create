# 手动处理验证码与继续运行说明

本文档说明在运行过程中遇到验证码或提示弹窗时，如何人工处理并继续执行。

## 适用场景

- 浏览器出现滑块验证码
- 提交后出现 `Got it` / 指南类弹窗，自动关闭失败
- 程序进入人工等待状态

## 当前机制

项目当前支持：

- 自动检测验证码/弹窗
- GUI 内提示更接近 Windows 10/11 风格的提醒通知
- 人工处理后点击“手动继续”恢复流程

项目当前不承诺：

- 对所有验证码场景都能自动绕过

## GUI 操作方式

1. 正常运行 `FlashSaleApp.exe` 或 `python app_gui.py`
2. 若界面右上角或日志提示需要人工处理验证码：
   - 切到浏览器
   - 手动完成验证码/关闭弹窗
3. 回到程序，点击【手动继续】
4. 程序收到继续信号后会恢复执行

## 文件方式继续

如果你不想点按钮，也可以在程序运行目录手动创建：

```text
.manual_resume.flag
```

程序检测到这个文件后也会继续执行。

## 常见日志提示

- `CAPTCHA detected. Please solve it manually in browser.`
  - 表示需要人工处理验证码

- `MANUAL PAUSE:`
  - 表示当前步骤进入人工等待

- `Waiting for manual continue...`
  - 表示程序正在等待你点击“手动继续”或创建标记文件

- `Manual continue detected.`
  - 表示程序已经收到继续信号

## 注意事项

- 不要在人工处理过程中随意切换 CDP 实例
- 如果已经处理完验证码，但程序仍未继续，请确认是否点击了【手动继续】
- 运行日志和运行截图默认不建议提交到公共仓库
